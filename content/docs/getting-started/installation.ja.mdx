# インストール・設定ガイド

本ガイドは二部構成です：
1. **テンプレートベースの開発**：既にプロジェクトを Fork または Clone している場合は、こちらをご覧ください。
2. **ゼロから構築**：プロジェクトがどのように構築されているか、ステップバイステップで理解したい場合は、後続のセクションをお読みください。

## 🚀 クイックスタート（テンプレートベース）

既に `git clone` でプロジェクトを取得済みの場合、以下の手順で環境を初期化してください。

### 1. 環境準備

プロジェクト環境の管理には Conda の使用を推奨します。Node.js バージョンの隔離が可能です（v20+ 推奨）。

```bash
# 独立環境を作成
conda create -n mota-atom nodejs=20 -y
conda activate mota-atom
```

### 2. 依存パッケージのインストール（重要）

プロジェクトを Clone した後、ビルドスクリプトに必要なツールライブラリ（`js-yaml`、`glob` など）を含むすべての依存パッケージを**最初に**インストールする必要があります。

> ⚠️ **よくあるエラー**：`npm run dev` を直接実行すると、`Cannot find package 'js-yaml'` というエラーが発生する場合があります。これはビルドスクリプトの依存パッケージがインストールされていないためです。

```bash
# 1. プロジェクトディレクトリに移動
cd mota-atom

# 2. すべての依存パッケージをインストール
npm install

# 3.（任意）スクリプト依存のエラーが解消しない場合、手動でインストール：
npm install -D js-yaml glob gray-matter dotenv
```

### 3. 環境設定

環境変数のサンプルファイルをコピーします：

```bash
cp .env.example .env.local
```
`.env.local` 内の各項目を設定してください（Supabase の URL と Key、メールサービス設定など）。

### 4. 開発サーバーの起動

```bash
npm run dev
```

起動に成功したら、`http://localhost:3000` にアクセスしてください。

## 🛠️ ゼロから構築ガイド（参考）

以下はプロジェクトの構築過程を記録したもので、参考資料としてのみお使いください。

## 1. 環境準備
Node.js バージョンを隔離するため、Conda による環境管理を推奨します。

```bash
# 独立環境を作成（Node.js 20 を指定）
conda create -n mota-atom nodejs=20 -y
```

# 環境を有効化
conda activate mota-atom

## 2. プロジェクト初期化
Next.js フレームワークと Shadcn UI コンポーネントライブラリを初期化します。

```bash
# 1. Next.js プロジェクトを初期化（TypeScript, Tailwind, ESLint）
npx create-next-app@latest mota-atom --typescript --tailwind --eslint

# プロジェクトディレクトリに移動
cd mota-atom

# 2. Shadcn UI を初期化
npx shadcn@latest init
💡 ヒント：Shadcn UI の設定時、カラーテーマは Slate または Gray を選択すると最適な視覚効果が得られます。
```

## 3. バックエンド統合（Supabase）
本プロジェクトでは、バックエンドサービス（データベース、認証）として Supabase を使用します。

### 3.1 CLI のインストールと初期化
```bash
# Supabase CLI をインストール（開発依存のみ）
npm install supabase --save-dev

# インストールを確認（バージョン番号が表示されるはずです。例：2.72.9）
npx supabase --version

# プロジェクトを初期化（supabase フォルダが生成されます）
npx supabase init
```

### 3.2 ローカル環境の起動（Local Docker）
Docker が実行中であることを確認してから、ローカル Supabase インスタンスを起動します：

```bash
npx supabase start
起動が成功すると、以下のローカルサービスアドレスが取得できます：
```

Console（Studio）：http://127.0.0.1:54323

Mailpit（メールテスト）：http://127.0.0.1:54323/project/default

### 3.3 依存パッケージのインストール
```bash
# @supabase/ssr はサーバーサイドコンポーネントやミドルウェアでの Cookie 処理に使用
npm install @supabase/supabase-js @supabase/ssr
```

### 3.4 環境変数の設定
プロジェクトルートディレクトリに .env.local ファイルを作成します：

```Code snippet
# npx supabase status から取得
NEXT_PUBLIC_SUPABASE_URL=http://127.0.0.1:54321
NEXT_PUBLIC_SUPABASE_ANON_KEY=あなたのローカルAnonKey
```

### 3.5 データベースワークフロー
開発段階：ローカルデータベースの構造を変更した後、以下のコマンドで SQL マイグレーションファイルを生成します：

```bash
# ローカルの変更を supabase/migrations ディレクトリに抽出
npx supabase db diff -f init_schema
npx supabase db diff -f add_user_table
```
本番同期：ローカルのテーブル構造の変更をリモート本番データベースにプッシュします：

```bash
# 事前に npx supabase login の実行が必要
npx supabase db push
```

## 4. コア機能

### 🌓 テーマ対応（ライト・ダークモード）
next-themes を統合して、ライト・ダークモードの切り替えに対応します。

```bash
npm install next-themes
```

設定：layout.tsx 内で ThemeProvider をラップしてください。Shadcn コンポーネントは自動的に適応します。

### 🌐 多言語対応（i18n）
next-intl を使用して国際化を実現します。

```bash
npm install next-intl
```
設定ファイル：src/i18n.ts、src/middleware.ts

翻訳ファイル：src/messages/zh.json、src/messages/en.json

ルーティング構造：src/app/[locale]/ の動的ルートを使用

### 📝 ブログシステム（Taxonomy テンプレート）
MDX と Tailwind Typography をベースにコンテンツエンジンを構築します。

```bash
# MDX パーサーとメタデータ処理ツールをインストール
npm install gray-matter next-mdx-remote
npm install -D @tailwindcss/typography
```
プラグイン設定：tailwind.config.ts を更新して typography プラグインを導入します。

読み取りエンジン：src/lib/mdx.ts

ページ実装：src/app/[locale]/(portal)/blog/page.tsx（Resend スタイルのリスト）

## 5. デプロイ・ビルド
本プロジェクトは Cloudflare Pages 向けに最適化されています。

### 5.1 アダプターのインストール
Cloudflare アダプターをインストールし、React のバージョンを固定します（互換性の確保）：

```bash
npm install next@15.5.2 react@19 react-dom@19 --save-exact
npm install -D @cloudflare/next-on-pages
```

### 5.2 プロジェクトのビルド
Cloudflare 専用コマンドでビルドを実行します：

```bash
npx @cloudflare/next-on-pages
```

### 5.3 ローカルプレビュー
GitHub にプッシュする前に、ローカルで Cloudflare Edge 環境をシミュレートしてテストします：

```bash
npx wrangler pages dev .vercel/output/static
```

## 付録：よく使うコマンド
- Supabase の状態確認：`npx supabase status`
- ローカルデータベースのリセット：`npx supabase db reset`
